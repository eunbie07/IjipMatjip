## 1. 전체 EPIC

> 사용자가 업로드한 방 사진을 기반으로 **MiDaS AI 모델**을 활용해 깊이 정보를 추정하고, 사용자가 선택한 4개의 점을 통해 **원근을 보정한** 실제 방의 **2D 평면도 및 3D 구조**를 자동으로 시각화하는 웹 서비스를 개발한다.
> 주요 기술 스택은 백엔드 **FastAPI**, 프론트엔드 **React(Vite), Tailwind CSS, React-Three-Fiber**이며, Docker/Kubernetes를 이용한 배포 자동화를 포함한다.

**EPIC-1: AI 모델 연동 및 핵심 API 개발 (완료)**
**EPIC-2: MiDaS 기반 원근 보정 및 크기 추정 로직 고도화 (완료)**
**EPIC-3: 2D/3D 결과 시각화 및 사용자 인터페이스(UI) 완성**
**EPIC-4: Docker 기반 배포 및 CI/CD 파이프라인 구축**

## 2. 스프린트 계획

### 2.1. 스프린트 1주차 (2025.07.08 ~ 07.12)

#### 2.1.1. 1주차 EPIC : AI 모델 연동 및 핵심 API 개발

> MiDaS 모델을 테스트하고, 업로드된 방 사진으로부터 깊이 맵(Depth Map)을 생성하는 FastAPI 서버를 구축한다. React(Vite) 기반의 프론트엔드 프로젝트를 초기 설정하고, 사진 업로드 및 층고 입력 UI를 구현한다.

#### 2.1.2. 일별 스크럼 (요약)

* **핵심 작업:**
  * **AI/백엔드:** MiDaS(small) 모델 로딩 및 추론 테스트, `/depth-map` API 구현 완료.
  * **백엔드:** FastAPI 서버 기본 구조 설계, CORS 설정, 이미지 처리를 위한 기본 엔드포인트 구성.
  * **프론트엔드:** Vite + React + Tailwind CSS 프로젝트 설정, `ImageUploader` 컴포넌트 구현, 건물 유형 선택 및 층고 입력 UI 구현.
* **성과:**
  * 사진을 업로드하면 백엔드에서 깊이 맵 이미지를 성공적으로 생성하고 프론트엔드로 전송하는 파이프라인 완성.
  * 견고한 컴포넌트 기반의 프론트엔드 구조 확립.

### 2.2. 스프린트 2주차 (2025.07.14 ~ 07.18)

#### 2.2.1. 2주차 EPIC : MiDaS 기반 원근 보정 및 크기 추정 로직 고도화

> MiDaS가 제공하는 상대적 깊이 정보를 활용하여, 사용자가 선택한 4개의 점으로부터 원근 왜곡을 보정한 실제 방 크기(가로, 세로)를 추정하는 핵심 로직을 개발한다. 또한, 측정 결과의 신뢰도를 계산하여 사용자에게 피드백을 제공하는 기능을 구현한다.

#### 2.2.2. 일별 스크럼 (요약)

* **핵심 작업:**
  * **백엔드:**
    * `/estimate-room-size` API 개발: 4개의 3D 좌표와 층고를 입력받아 원근을 보정한 방 크기(cm)를 반환.
    * **`improved_room_measurement` 함수 구현**: MiDaS 깊이 값의 표준편차와 범위를 활용하여 원근 보정 계수를 동적으로 계산하는 로직 적용.
    * **`calculate_confidence` 함수 구현**: 측정된 결과의 신뢰도를 기하학적 일관성, 깊이 값의 합리성 등을 기반으로 계산.
  * **프론트엔드:**
    * `ImageClickArea` 컴포넌트 개발: 사용자가 이미지 위에서 4개의 점을 클릭하고 좌표를 서버로 전송하는 기능 구현.
    * `axios`를 사용하여 모든 백엔드 API와의 연동 완료.
    * `@react-three/fiber` 라이브러리 추가 및 3D 뷰어(`RoomBox`) 기본 컴포넌트 설계.
* **성과:**
  * MiDaS의 한계를 극복하는 독자적인 원근 보정 및 신뢰도 계산 로직 구현 완료.
  * 프론트엔드-백엔드 간 데이터 통신 및 전체 측정 플로우 완성.

### 2.3. 스프린트 3주차 (2025.07.21 ~ 07.25)

#### 2.3.1. 3주차 EPIC : 2D/3D 결과 시각화 및 사용자 인터페이스(UI) 완성

> 백엔드로부터 받은 측정 결과를 활용하여, 2D 평면도와 3D 방 구조를 시각화하는 UI를 완성한다. 사용자가 결과를 직관적으로 이해할 수 있도록 탭(Tab)으로 2D/3D 뷰를 전환하는 기능을 구현하고, 측정 가이드 등 사용자 편의 기능을 개선한다.

#### 2.3.2. 일별 스크럼 계획

* **월요일 (7/21):**
  * **할 일:** 3주차 계획 리뷰, `RoomResult` 컴포넌트 개발 시작(2D 평면도), `RoomBox` 컴포넌트 개발 시작(3D 뷰어).
  * **목표:** 2D/3D 뷰 전환 탭 UI 구현, 측정된 가로/세로/높이 값을 3D 박스로 렌더링.
* **화요일 (7/22):**
  * **할 일:** 2D 평면도에 측정된 길이(cm)와 신뢰도 표시, 3D 뷰어에 OrbitControls(궤도 제어) 추가.
  * **목표:** 사용자가 3D 뷰를 회전/확대/축소하며 결과를 입체적으로 확인할 수 있도록 구현.
* **수요일 (7/23):**
  * **할 일:** `HelpTips` (측정 가이드) 모달 UI 개선, 로딩 및 에러 상태에 대한 피드백 강화.
  * **목표:** 서비스의 전체적인 사용자 경험(UX) 완성도 향상.
* **목요일 (7/24):**
  * **할 일:** (Stretch Goal) 사용자가 캔버스 위에서 측정된 평면도를 미세 조정하는 기능 기획.
  * **목표:** AI 측정의 한계를 보완할 수 있는 사용자 편집 기능의 프로토타입 구상.
* **금요일 (7/25):**
  * **할 일:** 3주차 작업 내용 통합 테스트 및 버그 수정, 4주차 배포 계획 구체화.
  * **목표:** 안정적인 시각화 기능 확보 및 다음 스프린트 준비.
